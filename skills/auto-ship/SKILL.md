# Auto-Ship — 自动交付 Pipeline

## 职责

当 auto-eval 通过后，自动执行：创建 MR → 监控 CI → 合并或退回。

## Pipeline 步骤

```
auto-eval PASS → Step 1: create MR → Step 2: wait CI → Step 3: merge/rollback
                         ↓ fail                    ↓ fail
                    通知 agent 修复            通知 xiaokang 人工介入
```

## Step 1: 自动创建 MR

```bash
git add [仅代码文件]
git commit -m "{type}({service}): {jira_key} — {summary}

Auto-generated by {agent_name}. Eval score: {score}/100.
Lint:{l}/20 Typecheck:{t}/20 Test:{p}/25 Coverage:{c}/15 Security:{s}/10"

git push origin {branch}

# 创建 MR (用 GitLab API 或 gh CLI)
# target=develop, remove_source=false, squash=false
```

## Step 2: 等待 CI

```bash
# 轮询 CI 状态 (每 60s)
while CI_status == "pending":
  sleep 60

if CI_status == "success":
  → Step 3 (合并)
elif CI_status == "failed":
  → 通知 agent 修复并重新 push
```

## Step 3: 自动合并 (可选，默认关闭)

```
⚠️ auto-merge 默认关闭，需 xiaokang 手动 approve
开启条件: eval ≥ 90 AND CI passed AND 非周五 16:00+
```

## Step 4: Jira 更新

```
jira_add_comment:

## 🚀 Auto-Ship 完成

| 阶段 | 状态 |
|------|:---:|
| Auto-Eval | ✅ {score}/100 |
| CI | ✅ passed |
| MR | {mr_url} |
| 合并 | {merged/pending review} |

jira_transition_issue → Ready for QA (id: 311)
```

## Hive HQ 上报

```bash
# Ship 开始
curl -s -X POST http://127.0.0.1:17711/report -H "Content-Type: application/json" -d '{
  "agentId":"auto-ship","type":"task_start",
  "title":"Ship-{jira_key}: Auto-delivery started"
}'

# Ship 完成
curl -s -X POST http://127.0.0.1:17711/report -H "Content-Type: application/json" -d '{
  "agentId":"auto-ship","type":"task_done",
  "title":"Ship-{jira_key}: Delivered",
  "summary":"MR: {mr_url} | CI: passed | Eval: {score}/100"
}'
```

## 安全约束

1. **不自动合并** — 始终需要人工 approve（除非 eval ≥ 95 + 非高峰时段）
2. **不 force push** — 永远不
3. **不跳过 CI** — CI 失败立即停止
4. **周五 16:00 后不部署** — 自动延迟到周一
