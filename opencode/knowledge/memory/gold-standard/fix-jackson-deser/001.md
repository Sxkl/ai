## Context
Error: `Unrecognized field "soId" not marked as ignorable` — Jackson's default `FAIL_ON_UNKNOWN_PROPERTIES=true` causes deserialization failure when the API returns fields not present in the entity model.

## Solution Pattern
```java
@JsonIgnoreProperties(ignoreUnknown = true)
public class ServiceResource {
    // existing fields
}
```

## Key Learnings
- Jackson's `ObjectMapper` defaults to FAIL_ON_UNKNOWN_PROPERTIES
- API response fields evolve faster than entity models — always ignore unknowns
- Apply at class level for all fields; use `@JsonProperty` for specific exclusions
- Same pattern applies to ALL entity classes receiving external JSON
