## Context
`NullPointerException` in `ForkJoinTask` — a null element in a collection causes NPE when processed via `.parallelStream()`.

## Solution Pattern
```java
// Before (crashes in ForkJoinPool)
list.parallelStream()
    .map(item -> process(item))
    .collect(Collectors.toList());

// After (null-safe)
list.stream()
    .filter(Objects::nonNull)
    .parallel()
    .map(item -> process(item))
    .collect(Collectors.toList());
```

## Key Learnings
- `parallelStream()` propagates to ForkJoinPool where NPE crashes are harder to debug
- `.filter(Objects::nonNull)` is the simplest guard; more explicit null handling may be needed
- Consider `.parallel()` instead of `.parallelStream()` for explicit parallel toggle
- If null elements are expected, use `Optional` in the source collection
