## Context
NPE from calling methods on potentially null objects with no defensive guard. Common in restTemplate response handling and chained getter calls.

## Solution Pattern
```java
// Before (crashes)
return restTemplate.postForObject(url, request, Response.class).getData();

// After (safe)
Response response = restTemplate.postForObject(url, request, Response.class);
if (response == null) {
    log.warn("Empty response from {}", url);
    return Collections.emptyList();
}
return response.getData();
```

## Key Learnings
- Always null-check external API responses before dereferencing
- Log the null case at WARN level for monitoring visibility
- Return safe defaults (empty list, Optional.empty()) instead of null
- Combine with K002 (avoid `e.getMessage()` in log) for comprehensive fix
