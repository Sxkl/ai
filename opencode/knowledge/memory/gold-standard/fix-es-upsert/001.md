## Context
`document_missing_exception` in Elasticsearch when update request targets a non-existent document. Default `docAsUpsert` is false.

## Solution Pattern
```java
UpdateRequest request = new UpdateRequest(index, docId)
    .doc(jsonMap)
    .docAsUpsert(true);  // ← key fix
```

## Key Learnings
- ES `UpdateRequest` fails on missing documents by default
- `.docAsUpsert(true)` switches to insert-if-not-exists behavior
- This is safe because ES document IDs are deterministic
- No risk of duplicate documents since upsert uses the same ID
