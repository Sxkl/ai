## Context
Normal business scenario (e.g., "user not found", "duplicate request") logged at ERROR level, polluting error monitoring with false positives.

## Solution Pattern
```java
// Before
log.error("User not found: {}", userId);

// After
log.warn("User not found: {}", userId);
```

## Key Learnings
- ERROR level should only be used for actionable system faults
- Expected business conditions: WARN or INFO
- This reduces alert fatigue and makes real errors visible in monitoring
- Always pair with a check: "Would I wake up at 3am for this?"
